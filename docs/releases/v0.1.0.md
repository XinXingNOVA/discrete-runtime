# Discrete Runtime v0.1.0

这是 Discrete Runtime 的首个公开版本，也是离散阶段调度模型的 Godot 4.6 / GDScript
参考实现。

Runtime 负责确定性的 Entry 调度、Effect/Fact 因果链、异步 Operation 边界、阶段出口
和生命周期状态。它不持有具体玩法状态，也不依赖 UI、场景树或某一种游戏类型。

## 安装

下载 `discrete-runtime-godot-0.1.0.zip`，将其中的：

```text
addons/discrete_runtime/
```

复制或合并到 Godot 项目的 `res://addons/` 下。Runtime 使用 `class_name` 提供类型，
不需要 Autoload，也不是 EditorPlugin。

当前正式验证基线为 Godot 4.6.3。

## 主要能力

- PhaseRequest、Operation、Effect、Fact、Result 与 Signal 六类 Entry。
- 分派前达到观察闭包。
- Effect 按较小优先级整数优先、同优先级按 `seq` 排序。
- Handler、Applier、Processor 与 ResultEmitter 按注册顺序工作。
- 同步 Effect/Fact 应用与异步 Operation 等待。
- `IMMEDIATE` 和 `AFTER_SETTLEMENT` 两种 Signal 出口模式。
- `NOT_STARTED`、`READY`、`ADVANCING`、`EXITED`、`TERMINATED`、`FAILED`
  六态生命周期。
- 并发 `advance()` 保护和明确失败信息。

## 验证

- Runtime 契约套件：21 个测试、94 个断言。
- GitHub Actions：Godot 4.6.3 Linux 无界面验证通过。
- addon-only ZIP 可以重复构建。
- 发布包已安装到全新临时 Godot 项目并完成最小运行闭环。
- 四类外部玩法案例在不修改 Runtime 源码的情况下完成验证。

完整玩法、UI、Offer/Commit、ContentPack、Projector、存档、网络和随机数策略均不属于
Runtime API。

## 版本状态

`v0.1.0` 表示核心契约已经具备可执行证据，但公共 API 尚未承诺 `1.0.0` 级别的长期
兼容性。后续不兼容调整必须由跨领域证据驱动，并记录在变更说明中。
